## Button Variants

`<Button />` components take in a `variant` property that defines the way the button should look and behave across the platform.

<Canvas>
  <Button variant='primary'>Primary</Button>
  <Button variant='outlined'>Outlined</Button>
  <Button variant='secondary'>Secondary</Button>
  <Button variant='subtle'>Subtle</Button>
  <Button variant='link'>Link</Button>
  <Button variant='destructive'>Destructive</Button>
  <Button variant='destructive-outlined'>Destructive Outlined</Button>
</Canvas>

## Buttons With Icon and Label

Occasionally you may want to have an icon accompany the label to enhance the UX and provide further context for the action associated with the button. As such you can pass in a `startIcon` and `endIcon` property which will automatically handle including the icon component. For example, you may want to have a close button that includes a cross icon.

<Canvas>
  <Button variant='destructive-outlined' startIcon={<DeleteIcon outlined />}>
    Delete
  </Button>
  <Button endIcon={<ArrowRightIcon />}>Next</Button>
</Canvas>

## Button Sizes

Not all buttons are created equal, and sometimes you may want to have a smaller or larger button depending on the context in which it's being used. For this reason you can pass a `size` property which will set the proper height and padding as necessary. By default buttons size to `md`.

<Canvas>
  <Button size='xs'>Button</Button>
  <Button size='sm'>Button</Button>
  <Button size='md'>Button</Button>
  <Button size='lg'>Button</Button>
</Canvas>
